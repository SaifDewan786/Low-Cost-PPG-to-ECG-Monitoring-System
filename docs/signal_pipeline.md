# Signal Pipeline

## Overview

The signal pipeline converts raw PPG recordings into model-ready input for ECG-like waveform reconstruction.

The pipeline is designed to support two input sources:

1. Benchmark paired PPG-ECG datasets
2. Custom PPG recordings from the low-cost wearable prototype

The benchmark branch supports supervised model training and evaluation. The custom branch supports real-device transfer testing.

## Signal Pipeline Summary

```text
Raw PPG signal
    ↓
Numerical parsing
    ↓
Sampling-rate preparation
    ↓
Windowing / segmentation
    ↓
Z-score normalization
    ↓
Clipping
    ↓
Moving average smoothing
    ↓
First derivative generation
    ↓
Second derivative generation
    ↓
Channel-wise normalization
    ↓
Stacking: PPG + dPPG + ddPPG
    ↓
TCN model input
```

## Sampling Preparation

The model-ready signal format uses 125 Hz preparation.

The benchmark signals were prepared in a model-compatible format, while custom-device recordings required additional conversion from raw text data into structured numerical form.

Correct sampling-rate interpretation is important. Incorrect sampling assumptions can distort waveform density and BPM estimation.

## Segmentation

The reconstruction model uses fixed-length signal windows.

The project used 1024-sample segments in the benchmark pipeline. At 125 Hz, this corresponds to approximately 8.19 seconds of signal.

This segment length is useful because it contains multiple cardiac cycles while remaining manageable for model training and inference.

## Normalization

Each signal segment is normalized using z-score normalization.

```text
normalized_signal = (signal - mean(signal)) / standard_deviation(signal)
```

A small epsilon value is used for numerical stability.

Normalization helps reduce scale differences between recordings and makes training more stable.

## Clipping

After normalization, signals are clipped to a fixed range.

The purpose of clipping is to reduce the influence of extreme outliers.

Common clipping range:

```text
[-5, 5]
```

This prevents rare abnormal values from dominating model input or loss behavior.

## Smoothing

Before derivative construction, the normalized PPG signal is mildly smoothed using a moving average filter.

The goal is not aggressive denoising. The goal is to reduce small high-frequency fluctuations so that derivative channels become more stable.

A representative smoothing choice is a small one-dimensional moving average window.

## First Derivative PPG

The first derivative of PPG is generated from the smoothed PPG signal.

The first derivative highlights:

* Rising slope
* Falling slope
* Pulse transition behavior
* Local rate of change
* Timing dynamics

This channel helps the model observe how the pulse waveform changes over time.

## Second Derivative PPG

The second derivative is generated by applying another gradient operation to the first derivative.

The second derivative highlights:

* Curvature
* Local acceleration
* Inflection-related structure
* Subtle pulse morphology
* Shape changes in the PPG waveform

This gives the model additional dynamic information beyond the raw PPG waveform.

## Multichannel Input

The final model input uses three channels:

| Channel | Meaning                            |
| ------- | ---------------------------------- |
| PPG     | Original normalized pulse waveform |
| dPPG    | First derivative of PPG            |
| ddPPG   | Second derivative of PPG           |

The final input format is:

```text
[PPG, dPPG, ddPPG]
```

This multichannel representation gives the reconstruction model access to waveform amplitude, slope, and curvature information.

## ECG Target in Benchmark Training

For supervised benchmark training, the target output is the corresponding ECG segment.

The ECG target is also normalized so that the model learns reconstruction in a standardized waveform range.

## Custom Device Signal Path

For custom wearable data:

1. PPG is recorded by the low-cost device.
2. The recording is saved as text data.
3. The text file is parsed into numerical time-series data.
4. The signal is prepared at 125 Hz model-ready format.
5. The same preprocessing and multichannel construction logic is applied.
6. The prepared signal is passed to the trained reconstruction model.

## Important Limitation

Custom-device recordings currently do not have paired simultaneously measured ECG ground truth.

Therefore, custom-device output can be inspected for:

* Rhythm consistency
* BPM similarity
* Signal plausibility
* Sensitivity to noise
* Transfer behavior

But it cannot yet be fully validated for ECG morphology accuracy.

## Summary

The signal pipeline is one of the most important parts of the system. It converts low-cost PPG recordings into a structured representation that can be used by the TCN reconstruction model.

The use of PPG, first derivative PPG, and second derivative PPG is a key design choice because it provides the model with complementary views of the same physiological signal.
